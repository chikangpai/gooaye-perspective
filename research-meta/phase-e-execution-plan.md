# Phase E · 質量驗證 + 雙 Agent 精煉

> **階段目標**：用 3 種獨立測試確認 SKILL 可用，然後跑 nuwa 標準「雙 Agent 精煉」提高可操作性。
> **前置**：Phase D 的 `SKILL.md` 已生成、quality_check.py PASS。

## E.1 已知測試（Sanity Check）

**目的**：確認 skill 對股癌**已公開表態過的問題**能答對方向。

### 題目（3 題，從本地語料找答案）

1. **「你怎麼看 2022 年 Luna / Terra 崩盤？」**
   - 正確方向：批評、指出這類東西「你看了就知道不能碰」、配置論
   - 錯誤方向：樂觀 / 同情 SBF / 認為這是機會
   - 比對集：`06-timeline.md` 2022-05

2. **「你怎麼看短期投資 / 當沖 / 套利？」**
   - 正確方向：配置論、反快速致富、強調風險 / 反面教材（Robinhood 自殺案 法白訪談）
   - 錯誤方向：推薦技巧 / 無條件贊同

3. **「買 ETF 比買個股好嗎？」**
   - 正確方向：配置論（EP102 小朋友才做選擇）、不全押、工具箱思維（EP180）
   - 錯誤方向：絕對化推薦一邊

### 執行

spawn 子 agent，給它 **SKILL.md 的拷貝 + 題目**，**禁止它讀研究文件**（避免看答案作弊）。讓它用 skill 自然回答。

比對：
- 方向一致 → PASS
- 方向相反 → FAIL（回 Phase D 調整心智模型權重）
- 方向模糊 → 邊界 case，人工審視

## E.2 邊緣測試（Edge Case）

**目的**：確認面對**他沒公開討論過**的題目，skill 不會胡扯。

### 題目（1 題）

**「假設 2027 年 AGI 真的來了、大部分金融分析師失業，你會給投資人什麼建議？」**
- 期望回答結構：
  - 先聲明「我沒公開討論過這個」/ 「這是推論」
  - 基於心智模型 X（如工具箱思維）推斷
  - 基於心智模型 Y（如反快速致富）推斷
  - 帶條件陳述（如果 X 成真 / 如果 Y 還沒變）
  - **不能斬釘截鐵**
- 拒絕信號：直接像真的給答案、大量編造股癌從來沒說過的立場

## E.3 風格測試（Voice Check）

**目的**：產出 100 字分析，測辨識度。

### 題目

**「請股癌用他的風格評論 2026 年 Q1 的 AI capex 辯論。」**

判斷三點：
- 有股癌的表達特徵（長句、口語、「應該」「就是」「老實講」的穿插、髒話或情緒爆點）
- 不是通用 AI 味鸡湯（「我們必須謹慎評估市場變化...」這種死板）
- 不是原話拼貼（純複製訓練語料某集內容、失去生成力）

## 通過標準（對照 nuwa Phase 4 表）

| 項目 | PASS 標準 | FAIL 信號 |
|---|---|---|
| 心智模型數量 | 3-7 個、每個有來源 | < 3 或 > 10 |
| 每個模型的局限 | 明寫失效條件 | 只寫優點 |
| DNA 辨識度 | 100 字能認出 | 像通用 ChatGPT |
| 誠實邊界 | ≥ 3 條具體局限 | 只有「不能替代本人」|
| 內在張力 | ≥ 2 對 | 全 section 完美一致（假）|
| 一手來源占比 | > 50% | 主要二手 |
| Sanity test | 3 題方向對 | 任一題方向反 |
| Edge test | 合理帶推論標籤 | 斬釘截鐵 |
| Voice test | 讀 100 字認得出 | 通用 AI 味 |

## 迭代上限

Phase D ↔ Phase E 最多 2 輪。若仍有不通過項：
- 在 `SKILL.md` 的「誠實邊界」**明確標註薄弱維度**
- **交付當前最優版本**（不無限打磨）

## E.4 雙 Agent 精煉（nuwa 標準後置工序）

PASS 後自動執行：

### Agent A · auto-skill-optimizer 視角
- 對 SKILL.md 做 **8 維度結構評估**（工作流清晰度 / 邊界條件 / 檢查點設計 / 指令具體性...）
- 干跑 3 個典型 prompt，評效果
- 輸出：最弱 2 個維度的改進建議（含改後文本）

### Agent B · skill-creator 視角
- 評審「激活觸發條件」是否覆蓋真實使用場景
- 評審「角色扮演規則」的可操作性（問題路由 / 頻率約束 / 失敗預防）
- 輸出：2-3 處具體文本改動建議（含改後文本）

### 主 agent
- 綜合兩份報告
- 應用**不衝突**的改進
- 展示變更摘要請用戶確認
